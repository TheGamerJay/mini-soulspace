# Mini SoulSpace — Development Phases

Mini SoulSpace is built phase by phase as a production-quality application.
Each phase must be production-ready, tested, modular, documented and
maintainable before the next begins.

## Phase 0 — Repository Foundation ✅ (current)

Establish the complete, clean, scalable foundation:

- GitHub integration
- Backend architecture (FastAPI, brains, config, Alembic)
- Frontend architecture (Next.js, App Router, landing page)
- AI architecture (prompts, model config, routing scaffolding)
- Docker (PostgreSQL + pgvector, Redis, Ollama)
- Setup scripts and documentation

## Future roadmap

- **Phase 1** — Authentication & user accounts.
- **Phase 2** — Diary writing & storage (entries CRUD).
- **Phase 3** — Reflection engine (the diary that talks back).
- **Phase 4** — Semantic memory (pgvector embeddings + recall).
- **Phase 5** — Emotional analytics, charts and timeline.
- **Phase 6** — SoulLibrary & curated collections.
- **Phase 7** — Safety & crisis-handling hardening.
- **Phase 8** — Realtime experience (WebSocket streaming).

> The roadmap is directional and refined as the product matures.
